# News AI — Congolese News RAG Assistant

A Retrieval-Augmented Generation (RAG) assistant for exploring and analysing
Congolese news articles. The pipeline parses a news dataset, splits articles
into chunks, embeds them into a [Chroma](https://www.trychroma.com/) vector
store using a local HuggingFace model, and answers natural-language questions
with an OpenAI chat model grounded in the retrieved articles.

## Features

- **Ingestion pipeline** — parse → chunk → embed a CSV of news articles.
- **Local embeddings** — `all-MiniLM-L6-v2` via `sentence-transformers` (no API cost).
- **Vector search** — semantic retrieval over a persistent Chroma database.
- **Grounded answers** — responses generated by an OpenAI model, framed as a
  historian-journalist analysing the retrieved sources (French output).
- **Two interfaces** — a Typer CLI (`cli.py`) and a Chainlit web chat (`app.py`).

## Architecture

```
data/actu_cd.csv
        │
        ▼
  NewsParser (src/parser.py)        load CSV into a DataFrame
        │
        ▼
  NewsChanker (src/chunker.py)      split article text into overlapping chunks
        │
        ▼
  NewsEmbedder (src/embed.py)       embed chunks → persist to chroma_db/
        │
        ▼
  NewsQA (src/query.py)             retrieve relevant chunks + ask the LLM
        │
        ├── cli.py    (terminal)
        └── app.py    (Chainlit web UI)
```

## Requirements

- Python 3.10+
- An OpenAI API key

## Installation

```bash
# 1. Clone and enter the project
git clone <your-repo-url>
cd News_AI

# 2. Create and activate a virtual environment
python -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate

# 3. Install dependencies
pip install -r requirements.txt
```

## Configuration

Copy the example environment file and add your OpenAI key:

```bash
cp .env.example .env
# then edit .env and set OPENAI_API_KEY=sk-...
```

> **Note:** `.env` is gitignored — never commit your API key.

## Data

The ingestion pipeline expects a CSV at `data/actu_cd.csv` with at least these
columns:

| Column           | Used as            |
|------------------|--------------------|
| `article_texts`  | document text      |
| `title`          | metadata: title    |
| `date`           | metadata: date     |
| `link`           | metadata: url      |

The dataset and the generated `chroma_db/` are **not** version-controlled
(they are large and rebuildable). Place your CSV in `data/` before running the
ingestion step.

## Usage

### 1. Build the vector store

```bash
python main.py
```

This parses the CSV, chunks the articles, embeds them, and persists the Chroma
database to `chroma_db/`. The full dataset takes a while and downloads the
embedding model on first run.

### 2. Ask questions

**Command line:**

```bash
python cli.py ask "Comment la situation avec les M23 a-t-elle évolué ?"
```

**Web interface (Chainlit):**

```bash
chainlit run app.py -w
```

Then open <http://localhost:8000> in your browser.
